What is 3DOM Classify?

3DOM Classify Viewer is an in-browser tool built on BabylonJS that lets you load point clouds (LAS, PLY, XYZ…), visually inspect and label them, train a Random-Forest classifier and then apply the trained model to new data — all without leaving the browser tab.

The application is served by a Django backend that handles file I/O and model training, while the frontend provides a real-time 3D viewport and interactive annotation tools.

Key Features

  • Real-time 3D point cloud rendering via BabylonJS (WebGL2)
  • Interactive rectangular and lasso selection tools
  • Training mode: annotate, configure features, run Random Forest
  • Classify mode: apply saved model to new point clouds
  • Measure and area-measure tools with on-screen labels
  • Multi-attribute coloring (RGB, intensity, classification…)
  • Export labelled data to LAS / CSV

Getting Started

Everything you need to run 3DOM Classify for the first time.

Prerequisites

Before installing, make sure your environment has:

  • Python ≥ 3.10
  • Django ≥ 4.2
  • A modern browser with WebGL2 support (Chrome 90+, Firefox 90+, Edge 90+)
  • At least 4 GB RAM (8 GB recommended for large point clouds)

Installation

Step-by-step guide to installing the application on your machine.

1 · Clone the Repository

bash git clone https://github.com/your-org/3dom-classify.git cd 3dom-classify

2 · Create a Virtual Environment

bash python -m venv .venv source .venv/bin/activate # Windows: .venv\Scripts\activate pip install -r requirements.txt

3 · Collect Static Files

BabylonJS and other static assets need to be placed in the correct directory:

bash python manage.py collectstatic

4 · Database Migrations

bash python manage.py migrate

5 · Run the Server

bash python manage.py runserver # → Open http://localhost:8000 in your browser
⚠️ Do not use runserver in production. Use Gunicorn + Nginx instead.
⚡ Quick Start

Quick Start

Load a point cloud and classify your first data in under 10 minutes.

Step 1 — Load a Point Cloud

Click File → Load Data in the top navbar, or press Ctrl+O. Select a supported file (LAS, PLY, XYZ). The processing pipeline will run and the point cloud will appear in the viewport.

Step 2 — Annotate in Training Mode

Switch to Training mode using the navbar toggle. Use the rectangle or lasso selection tool to select groups of points, then assign a class label from the right sidebar.

Step 3 — Train the Classifier

Once you have labelled points in at least two classes, open the Train section in the right sidebar, choose your features and click Run Training.

Step 4 — Classify

Switch to Classify mode. Load unlabelled data and click Run Classification. Results are coloured by predicted class in the viewport.

◫ User Guide

Interface Overview

Learn each area of the application UI.

Layout

The UI is split into four main areas: the Top Navbar, the Left Inspector panel, the Central Viewport, and the Right Tools panel. Each sidebar can be resized by dragging the divider between it and the viewport.

Top Navbar

The navbar contains the application logo, the File dropdown menu (Load / Download Data), and the Training / Classify mode toggle.

Central Viewport

The viewport renders the point cloud using BabylonJS. Controls:

  • Left click + drag — Rotate
  • Right click + drag — Pan
  • Scroll wheel — Zoom

The bottom info bar shows total point count, visible points, and current FPS. The top-left controls let you switch View and Color modes. The right floating toolbar holds the selection and measurement tools.

Left Panel — Inspector

The Inspector shows metadata about the loaded point cloud: point count, bounding box, available attributes, and active selection statistics.

Right Panel — Tools

Context-sensitive: in Training mode it shows class management and training configuration; in Classify mode it shows model selection and classification results.

📂 Load Data

Load Data

Supported file formats and how the loading pipeline works.

Supported Formats

  • .las / .laz — LiDAR industry standard
  • .ply — Stanford Polygon format
  • .xyz / .txt — plain-text ASCII point lists

Loading Pipeline

When you load a file, a multi-step pipeline runs on the backend and reports progress in real time:

1
Parse file

Read raw binary or ASCII data.

2
Extract attributes

XYZ, RGB, intensity, return number…

3
Compute features

Normals, local geometry descriptors.

4
Transfer to GPU

Upload vertex buffer to WebGL.

ℹ️ Files larger than 50 M points may take several minutes to load and require at least 8 GB RAM.
🎯 User Guide

Training Mode

Annotate point cloud data and train a Random Forest classifier.

Managing Classes

Open the right panel in Training mode. Use the + Add Class button to create labelled categories (e.g. Ground, Vegetation, Building). Each class gets a unique colour displayed in the viewport.

Selecting Points

Activate the Rectangle or Lasso tool from the floating right toolbar. Draw a selection on the viewport to highlight points. Hold Ctrl and draw to deselect points. Press I to invert the selection, Esc to clear it.

Assigning Labels

With a selection active, click a class badge in the right panel to assign that label to the selected points.

Feature Matrix

The Feature Matrix lets you choose which geometric features and at which neighbourhood radii are used during training. Toggle each cell to include or exclude a feature–radius combination.

Running Training

Click Run Training. A progress dialog shows pipeline steps. When complete, the model is saved automatically and becomes available in Classify mode.

Classify Mode

Apply a trained model to new, unlabelled point cloud data.

Select a Model

Switch to Classify in the navbar toggle. The right panel will list all previously trained models. Select the one you want to use.

Run Classification

Load your unlabelled point cloud via File → Load Data, then click Run Classification. Each point is assigned a class label and coloured accordingly in the viewport.

Review Results

Use the Color menu (top-left of viewport) to switch between Classification and other colour modes to compare results against raw attributes like RGB or intensity.

🛠 User Guide

Tools & Selection

All available viewport tools and how to use them.

Navigation (Default)

The default mode: rotate, pan and zoom the camera. No points are selected.

Rectangle Selection

Click and drag to draw a rectangle on screen. All visible points within the rectangle are selected. Hold Ctrl to deselect within the region.

Lasso Selection

Freehand selection. Click and drag to draw an irregular polygon. Useful for selecting complex-shaped regions.

Measure Tool

Click two points in the viewport to measure the 3D distance between them. The distance is displayed as an on-screen label.

Area Tool

Click multiple points to define a polygon. The computed planar area is shown in a cyan label. Double-click to close the polygon.

⬆ Reference

Export Data

Save your classified or annotated point clouds.

Export Formats

  • .las — preserves all attributes including the classification field
  • .csv — comma-separated X, Y, Z + class label per point

How to Export

Click File → Download Data in the top navbar. A dialog will ask you to choose the format and which attributes to include. Click Download to save the file to your machine.

⚠️ Exporting very large files may take a moment. Do not close the browser tab while the download is in progress.

FAQ

Frequently asked questions and troubleshooting tips.

The viewer is blank / black screen

Make sure your browser supports WebGL2. Visit chrome://gpu (Chrome) or about:support (Firefox) to verify hardware acceleration is enabled.

Loading large files is very slow

Files with tens of millions of points require significant processing time. Consider sub-sampling the cloud to a manageable density (e.g. 5 M points) before loading. Future versions will support progressive loading.

I can't find my trained model in Classify mode

Models are stored per-session. Make sure you haven't cleared your browser data or restarted the server. Models are saved to the media/models/ directory on the server.

The FPS counter shows very low values

High point counts stress the GPU. Try reducing the point size, hiding non-essential classes, or using the visibility filter to render only a subset of points.

How do I report a bug?

Open an issue on the project repository with a description of the problem, the browser and OS version, and the browser console output (F12).
