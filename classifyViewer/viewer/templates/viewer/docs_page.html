What is 3DOM Classify?

3DOM Classify Viewer is a local in-browser tool built on BabylonJS and Potree that lets you:

  • load point clouds or models (ply, las, glb);
  • visually inspect and label them;
  • train a Random-Forest classifier;
  • run inference on new, unlabelled data;

The application is served by a Django backend that handles file I/O and model training, while the frontend provides a real-time 3D viewport and interactive annotation tools.

📂 Load Data

Loading Data

How to load your point cloud dataset into the viewer.

Supported Formats

These are these supported formats for the viewer:

  • .ply — PLY Point Cloud;
  • .glb — GLB Model;
  • .las — LAS version 1.2 Format 3 Point Cloud;

How to Load a File

In the top left corner, click to File and then Load Data button. There will be a section like shown below:

Here it is possible to load from local a file and decide also to subsample the point cloud, in case of a PLY or LAS file. For the GLB format, it is possible to decide the target points of the point cloud.

Loading Pipeline

For semplicity and compatibility of the Potree library, the files are converted to a LAS version 1.2 Format 3 Point Cloud. Potree has the main task to prepare the point cloud data for BabylonJS.

Once ready the point cloud will be available for viewing and annotation.

🎯 Training Mode

Training Mode

Annotate point cloud data and train a model classifier.

Managing Classes

Now that the point cloud data is loaded, you can manage the classes for annotation. It is possible to translate and rotate the point cloud to help for the annotation process.

There are different type of point cloud selection methods available. They can be found in the Tools section.

When you select the point cloud to be annotated, you can assign it to a specific class with the right mouse button .

It is possible to change view from Color to Classification View to see the annotated classes.

Calculate Features

Later, you can calculate the features for your dataset. A detailed explanation can be found in the Features section

It is possible to decide from a list of available features and specify the radii to use for each feature.

Divide the dataset

Next step is to segment the dataset into training and validation sets. You can use the selection and the cut mode to divide the dataset.

Training sets must be around 70% of the total, while validation sets should be around 30%.

Training configuration

In the training configuration, you can decide the model name and the data split of the two sets as showned in the picture.

In addition, it is possible to choose which features to select during the training (RGB can be included using the button below)

Morever, you can decide the parameters values for RF model, which are:

  • n of estimators: number of trees in the forest (default: 200);
  • max depths: maximum depth of each tree (default: 15);
  • min samples split: minimum number of samples required to be at a split node (default: 20);
  • n of jobs: number of threads used to train the model (used only with CPU, default: 12);
  • use gpu: whether to use GPU for training you can flag the button (default: "off");

Training results

After training the model, a report will be given as output, as shown in the example below.

This viewer window explain the report with different sections:

  • metrics accuracy: overall accuracy of the model, precision, recall, F1-score, and IoU;
  • metrics accuracy per class: all metrics accuracy for each class
  • sorted feature importance: features ranked by their importance in the model; the sum of all the features should equal to 1.0. For instance, if you have 5 features, each feature will have a value between 0 and 1 (that represent the percentage of importance of that feature), and the sum of all 5 features will be 1.0. For simplicity, the report will visualize the most important features arriving to 0.91 and the other will be hidden, but the overall sum remain 1.0;
  • different processing time for each steps: loading data, training the model, computing metrics, and overall time;

🛠 Tools

Tools & Selection

All available viewport tools and how to use them.

Restore or reset settings

During the process, in case of some errors, there is the possibility to:

  • Restore the previous point cloud: a backup point cloud will always be available;
  • Restore deleted points: in case you delete points with CANC, a backup of the deleted points will always be available;
  • Reset the viewport scene: this allow to reset the scene to its initial state, be careful because the working directory will be cleaned;

Different view modes

There are different view modes available:

  • Perspective view: shows the point cloud with a perspective projection, simulating how the human eye perceives depth;
  • Orthographic view: shows the point cloud with an orthographic projection, where all lines remain parallel;
  • Top view (1 / Numpad 1): shows the point cloud from a top-down perspective;
  • Right view (2 / Numpad 2): shows the point cloud from a right-side perspective;
  • Front view (3 / Numpad 3): shows the point cloud from a front-facing perspective;
  • Left view (4 / Numpad 4): shows the point cloud from a left-side perspective;
  • Color view: displays the point cloud with colors based on intensity or other attributes;
  • Classification view: shows the point cloud with colors based on classification labels;
  • Scalar field view: displays the point cloud with colors based on scalar field values;

Tools

NB. Hovering the mouse over each tool for about one second will display a tooltip with the related controls and instructions.

Default mode

The default mode allows you to rotate, pan, and zoom the point cloud view using the mouse:

  • — Rotate
  • — Pan
  • — Zoom

Rectangle , Lasso and Polygon Selection

Rectangle is a selection tool that allows you to select points within a rectangular area.

On the other hand, lasso selection allows you to draw a freeform shape to select points within that shape.

Instead, polygon selection allows you to create a polygon by clicking to add vertices, and then close the shape to select points within it.

During the selection process, you can use the mouse and the keyboard to manipulate the selected points:

  • — for selecting points
  • CTRL +
    — to deselect points
  • — only for polygon selection, confirm the shape to select the points
  • Hold SHIFT — to select multiple areas
  • CANC — to delete selected points
  • I — to invert selection
  • Esc — to deselect all
  • / ENTER — polygon: add vertices and close
  • — to assign to a class available in the classes list on the left

Cut , Frame Selection and Merge

The cut mode is useful to isolate specific parts of the point cloud for detailed inspection or editing. After selecting a region with the frame tool, you can use the keyboard shortcut C to remove the selected points.

The frame selection tool allows you to move the camera view towards to the selected region. The keyboard shortcut F can be used for this purpose.

If you want to merge multiple segments in the outline sections (on the left panel), firstly hold SHIFT and click on the segments you want to merge, then with the right mouse button will appear the merge option.

Translation and Rotation

If the point cloud is imported in wrong way, you can translate or rotate it to the correct position. Gizmo reference is showned in bottom left corner to understand the orientation of the axis.

To translate the point cloud, you can use the keyboard shortcut T. To rotate the point cloud, you can use the keyboard shortcut R, then if you want to snap the rotation to 10 degree increments, you can hold SHIFT and the left mouse button .

Measure Tool and Area Measurement

Two measurement modes are available: distance measurement and area measurement. In the first, you can first click the point A and then second click the point B to measure the distance between them.

For area measurement, you can click on multiple points to define the area and to calculate the total area just click again to the first point for closing the shape.

Viewport settings

On the right panel you can adjust the viewport settings:

  • Show Grid: toggle the visibility of the grid on the ground plane (default ON);
  • Light Background: toggle the light background color (default OFF);
  • Point Size: adjust the size of the points in the point cloud;

✦ Features

Features

Geometric features used by the classifier and how to configure them.

What Are Features?

Geometric features are quantitative measures that describe the shape and structure of point cloud data. They capture local geometric properties of the point cloud, such as curvature, density, and spatial distribution. These features are used by machine learning algorithms to classify points into different categories based on their geometric characteristics. During the training phase, the viewer computes a feature matrix for the annotated points, which is then used to train the Random Forest classifier. The features helps the model to learn patterns and relationships in the data, enabling it to make accurate predictions on new, unlabelled point clouds during the classification phase.

Available Features

For each feature, provide a brief description of what it represents and how it is computed:

  • Anisotropy: Measures the degree to which the point cloud is elongated along one axis.
  • Omnivariance: A measure of the overall variance in all directions.
  • Sphericity: Indicates how spherical the point cloud is.
  • Planarity: Reflects the extent to which the point cloud lies on a plane.
  • Linearity: Measures the extent to which the point cloud is aligned along a line.
  • Verticality: Indicates the degree to which the point cloud is oriented vertically.
  • Surface Variation: Captures changes in surface orientation within the point cloud.
  • Neighbours: The number of neighbouring points considered in feature computation.
  • Vertical Range: The difference in height between the highest and lowest points in the neighbourhood.
  • Height Above: The average height of points above a reference plane.
  • Height Below: The average height of points below a reference plane.
  • Height: The absolute height of each point relative to a reference plane.

Decide the Radii

During the feature calculation phase, the choice of neighbourhood radius is crucial for capturing relevant geometric properties. A smaller radius may miss broader patterns, while a larger radius can smooth out important details. It is recommended to experiment with different radius values to find the optimal balance for your specific dataset and classification task. In the viewer panel there is the possibility to specify 1 to 5 seaching radii in meters (e.g., 0.5, 1.0, 2.0) for the features. The viewer will compute the features for each radius and then the model will decide which one to use during the training phase based on the feature list.

Calculate Features

When you click the Calculate Features button, you will see a panel like the one below. You can decide which features to compute (using the flags) and specify the neighbourhood radii. The viewer will compute all selected features for each point in the point cloud using the specified neighbourhood radii. In the option there is the possibility to use the GPU for accelerate the process.

🔬 Classify Mode

Classify Mode

Apply a trained model to new, unlabelled point cloud data.

Classification Configuration

For the classification mode, you have to click the Classify button on the top left. In this panel you have the possibility to load another dataset point cloud from local storage or use the previous one from training. In case if you have a model downloaded in local you can uploade it. N.B. the model zip that you want to upload must contain report.txt, model.pkl, and metadata.json files, like the one you can download from viewer after training, otherwise it won't be accepted.

Once you upload the test dataset, you can click to Start Classify button and you will have a panel like the one below.

Here you can select which input point cloud and which pre-trained model to use for the classification. You can also decide to use the GPU to accelerate the classification process. When you start the classification, the viewer will compute the features for the new point cloud and then it will apply the trained model to predict the classes for each point.

Classification results

Once the classification step is completed, you can visualize the results using the prediction view.
